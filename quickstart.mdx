---
title: "Quickstart"
description: "Install prerequisites, configure Connector, and send your first CSV load request in minutes."
---

<Steps>
  <Step title="Install prerequisites">
    Connector requires JDK 17 and Leiningen 2.0.0 or later.

    **Verify Java:**

    ```bash
    java -version
    ```

    The output must show major version `17`. Example:

    ```text
    openjdk version "17.0.11" 2024-04-16
    OpenJDK Runtime Environment (build 17.0.11+9)
    ```

    **Install Leiningen:**

    Follow the instructions at [https://github.com/technomancy/leiningen](https://github.com/technomancy/leiningen), then verify:

    ```bash
    lein version
    ```

    Expected output: `Leiningen 2.x.x on Java 17 ...`
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Jay17423/connector.git
    cd connector
    ```
  </Step>

  <Step title="Configure the service">
    Application configuration is loaded from `config/config.edn` at startup. Create or edit the file with your Spark settings:

    ```edn config/config.edn
    {:server {:port 3000}
     :spark  {:app {:name       "connector-app"
                    :master-url "local[*]"}}}
    ```

    | Key | Description | Default |
    |---|---|---|
    | `server.port` | HTTP server port | `3000` |
    | `spark.app.name` | Spark application name | required |
    | `spark.app.master-url` | Spark master URL | required |

    <Note>
      Use `local[*]` as the master URL for local development. For a Spark cluster, set this to your master address, e.g., `spark://host:7077`.
    </Note>
  </Step>

  <Step title="Build and run">
    From the project root, build the uberjar and start the service:

    ```bash
    lein uberjar
    lein run
    ```

    When startup is complete, the API is available at `http://localhost:3000`.

    <Info>
      The first build may take several minutes as Leiningen downloads Spark and its dependencies.
    </Info>
  </Step>

  <Step title="Send a test request">
    Use `curl` to load a local CSV file:

    ```bash
    curl -X POST http://localhost:3000/data-load \
        -H "Content-Type: application/json" \
        -d '{
            "type": "local",
            "format": "csv",
            "link-type": "public",
            "source": {
                "path": "/data/input.csv"
            },
            "options": {
                "header": true,
                "delimiter": ","
            }
        }'
    ```

    Replace `/data/input.csv` with the absolute path to a CSV file on your machine.
  </Step>

  <Step title="Review the response">
    A successful response returns HTTP `200` with the following structure:

    ```json
    {
        "status": "success",
        "source": "local",
        "duration-ms": 123,
        "data": [
            {
                "columnA": "value1",
                "columnB": "value2"
            }
        ]
    }
    ```

    | Field | Description |
    |---|---|
    | `status` | `"success"` on success, `"error"` on failure |
    | `source` | The `type` value from your request |
    | `duration-ms` | Total time to load and return the data |
    | `data` | Array of up to 10 rows, with column names as keys |

    If the request body fails validation, you will receive HTTP `400`:

    ```json
    {
        "status": "error",
        "msg": "Invalid request body",
        "error": "...spec explain output...",
        "duration-ms": 15
    }
    ```
  </Step>
</Steps>

## Next steps

<Tip>
  Connector supports five source types. Once your local setup is working, try loading files from cloud storage:

  - [Google Drive](/sources/google-drive) — public shared links or private files with OAuth2
  - [Dropbox](/sources/dropbox) — public links or private paths with an access token
  - [Amazon S3](/sources/amazon-s3) — public or private buckets with IAM credentials
  - [Google Cloud Storage](/sources/google-cloud-storage) — public or private buckets with a service account
</Tip>
