---
title: 'Authentication'
description: 'How to obtain and provide credentials for each supported data source.'
---

Connector supports public and private access to data sources. For public files, authentication fields can be omitted. For private files, you must include the relevant credentials in the `auth` object of your request body.

## Public vs private access

Every request includes a `link-type` field that controls whether authentication is required.

| `link-type` | Description |
|---|---|
| `"public"` | The file is publicly accessible. `auth` fields can be omitted. |
| `"private"` | The file requires credentials. The relevant `auth` fields are validated server-side and must be present. |

If `link-type` is omitted, it defaults to `"public"`.

## Provider credentials

<Tabs>
  <Tab title="Google Drive">
    Google Drive private access uses OAuth2. Connector exchanges your refresh token for a short-lived access token automatically on each request.

    **Required credentials**

    | Field | Location in request | Description |
    |---|---|---|
    | `client-id` | `auth.client-id` | OAuth2 client ID from Google Cloud Console |
    | `client-secret` | `auth.client-secret` | OAuth2 client secret |
    | `refresh-token` | `auth.refresh-token` | Long-lived refresh token obtained during OAuth2 authorization |

    **How to obtain credentials**

    1. Open [Google Cloud Console](https://console.cloud.google.com) and create or select a project.
    2. Go to **APIs & Services > Library** and enable the **Google Drive API**.
    3. Go to **APIs & Services > Credentials** and create an **OAuth 2.0 Client ID** (application type: Desktop or Web).
    4. Download the client credentials JSON. Your `client-id` and `client-secret` are in this file.
    5. Run the OAuth2 authorization flow (e.g., using the Google OAuth Playground or a local script) to obtain a `refresh-token`.

    <Warning>
      The refresh token must have been granted the `https://www.googleapis.com/auth/drive.readonly` scope. Tokens with insufficient scope will cause download failures.
    </Warning>

    **Example request body (private)**

    ```json
    {
      "type": "gdrive",
      "format": "csv",
      "link-type": "private",
      "source": {
        "url": "https://drive.google.com/file/d/FILE_ID/view?usp=sharing"
      },
      "auth": {
        "client-id": "your-client-id",
        "client-secret": "your-client-secret",
        "refresh-token": "your-refresh-token"
      },
      "options": {
        "header": true,
        "delimiter": ","
      }
    }
    ```

    **Example request body (public)**

    ```json
    {
      "type": "gdrive",
      "format": "csv",
      "link-type": "public",
      "source": {
        "url": "https://drive.google.com/file/d/FILE_ID/view?usp=sharing"
      },
      "options": {
        "header": true,
        "delimiter": ","
      }
    }
    ```
  </Tab>

  <Tab title="Dropbox">
    Dropbox private access uses a long-lived API token. Public Dropbox links do not require any credentials.

    **Required credentials**

    | Field | Location in request | Description |
    |---|---|---|
    | `token` | `auth.token` | Dropbox API access token |

    **How to obtain a token**

    1. Go to [https://www.dropbox.com/developers/apps](https://www.dropbox.com/developers/apps) and create a new app.
    2. Select **Scoped access** and choose the appropriate permissions (at minimum, `files.content.read`).
    3. Under the **Settings** tab of your app, click **Generate** to create an access token.

    <Note>
      For public Dropbox links, use `source.url`. For private files accessed with a token, use `source.path` (the path within your Dropbox, e.g., `/reports/data.csv`).
    </Note>

    **Example request body (private)**

    ```json
    {
      "type": "dropbox",
      "format": "csv",
      "link-type": "private",
      "source": {
        "path": "/reports/data.csv"
      },
      "auth": {
        "token": "your-dropbox-token"
      },
      "options": {
        "header": true,
        "delimiter": ","
      }
    }
    ```

    **Example request body (public)**

    ```json
    {
      "type": "dropbox",
      "format": "csv",
      "link-type": "public",
      "source": {
        "url": "https://www.dropbox.com/s/FILE_ID/data.csv?dl=0"
      },
      "options": {
        "header": true,
        "delimiter": ","
      }
    }
    ```
  </Tab>

  <Tab title="Amazon S3">
    Amazon S3 private access uses AWS IAM credentials. Public S3 objects (buckets or objects with public read ACLs) do not require credentials.

    **Required credentials**

    | Field | Location in request | Description |
    |---|---|---|
    | `access-key` | `auth.access-key` | AWS IAM access key ID |
    | `secret-key` | `auth.secret-key` | AWS IAM secret access key |

    **How to obtain credentials**

    1. Open the [AWS IAM Console](https://console.aws.amazon.com/iam).
    2. Create or select an IAM user.
    3. Under **Security credentials**, generate an access key.
    4. Copy the **Access key ID** and **Secret access key**.

    <Tip>
      Use an IAM policy with minimal permissions. The only action required is `s3:GetObject` scoped to the specific bucket and key. Avoid using root credentials or overly broad policies.
    </Tip>

    **Example request body (private)**

    ```json
    {
      "type": "s3",
      "format": "csv",
      "link-type": "private",
      "source": {
        "bucket": "my-bucket",
        "key": "data/transactions.csv",
        "region": "us-east-1"
      },
      "auth": {
        "access-key": "AKIAIOSFODNN7EXAMPLE",
        "secret-key": "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"
      },
      "options": {
        "header": true,
        "delimiter": ","
      }
    }
    ```

    **Example request body (public)**

    ```json
    {
      "type": "s3",
      "format": "csv",
      "link-type": "public",
      "source": {
        "bucket": "my-public-bucket",
        "key": "data/transactions.csv",
        "region": "us-east-1"
      },
      "options": {
        "header": true,
        "delimiter": ","
      }
    }
    ```
  </Tab>

  <Tab title="Google Cloud Storage">
    Google Cloud Storage private access uses a GCP service account. Connector constructs a service account JSON key from the credentials you provide and configures Spark accordingly.

    **Required credentials**

    | Field | Location in request | Description |
    |---|---|---|
    | `project-id` | `auth.project-id` | GCP project ID |
    | `client-email` | `auth.client-email` | Service account email address |
    | `private-key` | `auth.private-key` | Service account private key (PEM format) |

    **How to obtain credentials**

    1. Open the [GCP Console](https://console.cloud.google.com) and select your project.
    2. Go to **IAM & Admin > Service Accounts** and create a new service account.
    3. Grant the service account the **Storage Object Viewer** role on the target bucket.
    4. Under the service account's **Keys** tab, add a key and select **JSON**. Download the key file.
    5. Extract `project_id`, `client_email`, and `private_key` from the JSON file.

    <Note>
      The `private_key` field in the downloaded JSON contains literal `\n` sequences. When passing the key as a JSON string value in the request body, you can use `\\n` — Connector automatically converts `\\n` to actual newlines before configuring Spark.
    </Note>

    **Example request body (private)**

    ```json
    {
      "type": "gcs",
      "format": "csv",
      "link-type": "private",
      "source": {
        "bucket": "my-gcs-bucket",
        "key": "data/file.csv"
      },
      "auth": {
        "project-id": "my-gcp-project",
        "client-email": "svc-account@my-gcp-project.iam.gserviceaccount.com",
        "private-key": "-----BEGIN PRIVATE KEY-----\\nMIIE...\\n-----END PRIVATE KEY-----\\n"
      },
      "options": {
        "header": true,
        "delimiter": ","
      }
    }
    ```

    **Example request body (public)**

    ```json
    {
      "type": "gcs",
      "format": "csv",
      "link-type": "public",
      "source": {
        "bucket": "my-public-gcs-bucket",
        "key": "data/file.csv"
      },
      "options": {
        "header": true,
        "delimiter": ","
      }
    }
    ```
  </Tab>
</Tabs>
